<objective>
Create the `src/mutation_strategies/` namespace package by splitting the current monolithic `src/mutations.py` (132 lines mixing registry I/O, bandit logic, and API façade) into three single-responsibility modules, and reduce cyclomatic complexity of `MutationBandit.select()` via small helper functions (NOT OO patterns, per CONTEXT.md D-02).

Purpose: ARC-03 densification (D-01) + ARC-02 complexity reduction (D-02). The old `src/mutations.py` becomes a re-export shim in plan 01-05, so existing `from src.mutations import registry, MutationBandit, ...` imports keep working.
Output: Three new files under `src/mutation_strategies/`.

**DEVIATION NOTE (load-bearing):** 01-PATTERNS.md names the new package `src/mutations/`. That is impossible in Python: a module file (`src/mutations.py`) and a same-named directory cannot coexist — the file shadows the directory and `from src.mutations.registry import ...` raises `ImportError` (verified empirically). The new package is therefore named `src/mutation_strategies/`. This honors locked decision D-01 ("create logical subfolders with small files") and the repo invariant (zero `__init__.py` files). The old `src/mutations.py` is kept as the backward-compat shim in plan 01-05, exactly as PATTERNS intends for that file.
</objective>
