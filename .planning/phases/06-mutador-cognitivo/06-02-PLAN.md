# Plan 06-02 — Optimizer: Routing Intercept e Prior Boosting

## Goal

Integrar `MutadorCognitivoAgent` ao fluxo de execução do MCTS:
1. Instanciar `self.agent_cognitivo = dspy.ChainOfThought(MutadorCognitivoAgent)` em `Optimizer.__init__`.
2. Injetar prior boosting para `mutador_cognitivo` via `self.mutation_bandit.load_priors()` usando os valores de config.
3. Interceptar o chamado de agente em `_expand_node()` para rotear para `self.agent_cognitivo` quando `strategy == 'mutador_cognitivo'`, com validação pós-chamada de `raciocinio_estruturado`.

## Context

Este plano depende do 06-01 (Signature e config já existem). A tarefa mais delicada é a modificação cirúrgica de `_expand_node()` — o método tem 100 linhas com retry loop, o branch `__DISCOVER__`, experience context, value estimator pruning e backprop. O intercept deve acontecer **apenas** no bloco de chamada do agente (linha ~292) e não deve alterar nenhuma lógica antes ou depois desse ponto.

A validação de `raciocinio_estruturado` é **soft enforcement**: log de erro via `self.on_error(...)` mas sem interromper a expansão — a `nova_instrucao` ainda é aceita mesmo que o raciocínio seja mal-formatado na primeira iteração.

<action>
Em `_expand_node()`, substituir APENAS o bloco da chamada do agente (`predicao = self.agent(...)` com seus argumentos) por um bloco condicional:

Constante local `COGNITIVO_KEY = 'mutador_cognitivo'`.

Branch 1 (`if strategy == COGNITIVO_KEY:`):
- Chamar `self.agent_cognitivo(instrucao_anterior=leaf.instruction, nota_anterior=nota, feedback_juiz=feedback_completo, estrategia_mutacao=strategy_prompt)`.
- Atribuir resultado a `predicao`.
- Logo após, tentar `_validate_raciocinio(predicao.raciocinio_estruturado)` em try/except: se levantar Exception, chamar `self.on_error(f'[!] raciocinio_estruturado inválido: {e}')` e prosseguir (soft enforcement — não interromper o loop).
- Atribuir `critica = predicao.critica` e `nova_instrucao = predicao.nova_instrucao` usando as mesmas variáveis que o código original usa após a chamada.

Branch 2 (`else:`):
- Chamada original `self.agent(instrucao_anterior=..., nota_anterior=..., feedback_juiz=..., estrategia_mutacao=...)`.
- Atribuir resultado a `predicao`.
- Atribuir `critica = predicao.critica` e `nova_instrucao = predicao.nova_instrucao`.

O código APÓS o bloco condicional (candidata extraction, pruning, break etc.) permanece COMPLETAMENTE INALTERADO — ele usa as variáveis `predicao.nova_instrucao`, `predicao.critica` ou as locais `nova_instrucao`/`critica` já definidas no fluxo original.

NÃO modificar: o bloco `__DISCOVER__`, `strategy_prompt`/`strategy_desc` extraction, experience context, o retry loop exterior (`for tentativa in range(3)`), o `_should_prune`, o `backpropagation`, nem o `MCTSNode` construction.
</action>
