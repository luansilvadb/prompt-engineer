# Plan 06-01 — Signature, Validadores Pydantic, Config e Registry Seed

## Goal

Criar os artefatos fundacionais do MutadorCognitivo: a DSPy Signature `MutadorCognitivoAgent` com todos os campos redeclarados explicitamente, o Pydantic model `RaciocinioCognitivo` com validators obrigatórios, as duas chaves de config em `get_mcts_config()`, e o método `_seed_hardcoded_strategies()` no `StrategyRegistry`.

## Context

Esta fase entrega o requisito COGN-01 em camadas. Este plano (Wave 1) estabelece a fundação estática — tipos, validadores e registro — que os planos subsequentes consomem. Nenhuma chamada de LLM é feita aqui; todas as adições são puramente definicionais e idempotentes.

Riscos críticos documentados em 06-RESEARCH.md:
- DSPy Signature inheritance é instável — TODOS os campos de `SelfReflectiveAgent` devem ser redeclarados explicitamente em `MutadorCognitivoAgent`.
- Pydantic validators NÃO disparam automaticamente em DSPy Predictions — `RaciocinioCognitivo` é um modelo independente para validação pós-chamada.
- `_seed_hardcoded_strategies()` deve ser idempotente via guard `if 'mutador_cognitivo' not in self.strategies`.
